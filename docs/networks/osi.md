# Модель OSI

Эталонная модель взаимодействия открытых систем (Open Systems Interconnection) — 7-уровневая архитектура сетевых протоколов.

## Уровни модели OSI

### 7. Прикладной (Application)

Интерфейс между сетью и приложениями пользователя.

| Протокол | Порт | Назначение |
|----------|------|------------|
| HTTP | 80 | Веб-страницы |
| HTTPS | 443 | Защищённый веб |
| FTP | 20, 21 | Передача файлов |
| SSH | 22 | Удалённый доступ |
| SMTP | 25, 587 | Отправка почты |
| DNS | 53 | Разрешение имён |
| DHCP | 67, 68 | Автоконфигурация |

### 6. Представления (Presentation)

Преобразование форматов данных.

**Функции:**
- Шифрование/дешифрование (SSL/TLS)
- Сжатие данных
- Преобразование кодировок (ASCII ↔ EBCDIC)
- Форматы: JPEG, MPEG, GIF

### 5. Сеансовый (Session)

Управление сеансами связи.

**Функции:**
- Установление и завершение соединений
- Синхронизация
- Контрольные точки

**Примеры:** NetBIOS, RPC, PPTP

### 4. Транспортный (Transport)

Надёжная доставка данных.

| Протокол | Характеристики |
|----------|----------------|
| **TCP** | Надёжный, с установлением соединения, гарантия доставки |
| **UDP** | Ненадёжный, без соединения, быстрый |

**PDU:** Сегмент (TCP) / Датаграмма (UDP)

### 3. Сетевой (Network)

Логическая адресация и маршрутизация.

**Основные протоколы:**
- **IP** — адресация и маршрутизация
- **ICMP** — диагностика (ping, traceroute)
- **ARP** — связь IP ↔ MAC
- **OSPF, BGP, RIP** — протоколы маршрутизации

**PDU:** Пакет

**Оборудование:** Маршрутизатор (Router)

### 2. Канальный (Data Link)

Физическая адресация и контроль ошибок.

**Подуровни:**
- **LLC** (Logical Link Control) — взаимодействие с сетевым уровнем
- **MAC** (Media Access Control) — доступ к среде

**PDU:** Кадр (Frame)

**Оборудование:** Коммутатор (Switch), Мост (Bridge)

### 1. Физический (Physical)

Передача битов по физической среде.

**Характеристики:**
- Напряжения и токи
- Типы разъёмов
- Схемы кодирования

**PDU:** Биты

**Оборудование:** Хаб, Кабели, Разъёмы, Медиаконвертеры

## Инкапсуляция данных

При передаче данные проходят **инкапсуляцию** — добавление заголовков на каждом уровне:

```
┌─────────────────────────────────────────────────────────┐
│ Приложение: Данные                                      │
├─────────────────────────────────────────────────────────┤
│ Транспорт: [TCP Header] + Данные = Сегмент              │
├─────────────────────────────────────────────────────────┤
│ Сеть:      [IP Header] + Сегмент = Пакет                │
├─────────────────────────────────────────────────────────┤
│ Канал:     [Eth Header] + Пакет + [CRC] = Кадр          │
├─────────────────────────────────────────────────────────┤
│ Физический: 01101001011010010...                        │
└─────────────────────────────────────────────────────────┘
```

## Сравнение OSI и TCP/IP

| OSI | TCP/IP | Протоколы |
|-----|--------|-----------|
| 7. Прикладной | Прикладной | HTTP, FTP, DNS |
| 6. Представления | ↑ | SSL/TLS |
| 5. Сеансовый | ↑ | NetBIOS |
| 4. Транспортный | Транспортный | TCP, UDP |
| 3. Сетевой | Сетевой | IP, ICMP |
| 2. Канальный | Канальный | Ethernet |
| 1. Физический | ↑ | — |
